# Implementation Plans

Los planes se usan despues de aceptar una spec y antes de codificar.

Un plan debe dividir el trabajo en pasos pequenos, seguros y verificables. Debe ayudar a que Codex implemente sin mezclar refactors innecesarios ni romper el MVP.

Un buen plan incluye:

- Spec relacionada.
- Archivos o areas a tocar.
- Pasos de implementacion.
- Tests a agregar o actualizar.
- Validaciones manuales.
- Riesgos o puntos donde detenerse.

Los planes no reemplazan a las specs: bajan una spec aprobada a una secuencia concreta de trabajo.
