# Workflows

Use these workflows to choose the right path before changing the project.

- `spec-driven-development.md`: for ideas, field feedback, or meaningful product changes that need a spec before implementation.
- `feature-implementation.md`: for implementing an approved spec or a clearly scoped feature slice.
- `bugfix.md`: for correcting broken behavior while protecting the stable MVP.
- `field-testing-feedback.md`: for turning practice or tournament feedback into clear bugs, specs, and priorities.

All workflows must follow `docs/constitution.md`, relevant specs, plans, decisions, and applicable agent role guides.
