# Field Testing Feedback Workflow

Use this after practice, tournament, or real device testing.

1. Capture feedback with the date, device, match context, and who reported it.
2. Separate bugs from product improvements.
3. Convert important improvements into specs before implementation.
4. Prioritize changes that improve live usability, speed, clarity, and coaching decisions.
5. Avoid overloading the match recording flow.
6. Add manual QA notes to the relevant checklist or spec.
7. Update `docs/implementation-log.md` when feedback leads to meaningful changes.
