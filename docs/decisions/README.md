# Decision Log

Este directorio guarda decisiones tecnicas o de producto importantes.

Usar decision records livianos cuando una decision pueda afectar futuras features, arquitectura, datos o UX.

Ejemplos de decisiones a documentar:

- Por que la app es offline-first.
- Por que todavia no hay backend.
- Por que la ubicacion de caida del punto debe ser explicita.
- Por que los resumenes deben usar mapas o visualizaciones y no solo numeros crudos.
- Por que se agrega o evita una dependencia importante.

Usar `000-template.md` como base.
